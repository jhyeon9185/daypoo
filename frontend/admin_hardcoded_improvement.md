# 관리자 페이지 하드코딩 개선 가이드 (프론트엔드)

> 작성일: 2026-03-31

---

## 현황 요약

관리자 페이지 내 일부 데이터가 하드코딩 또는 더미 데이터로 되어 있어, 실제 API 연동이 필요합니다.
아래는 **프론트엔드 기준** 수정이 필요한 항목과 개선 방향입니다.

---

## 1. DashboardView (대시보드)

| 항목 | 현재 상태 | 위치 (줄) | 개선 방향 |
|---|---|---|---|
| 현재 접속자 | 초기값 342, 3초마다 ±3 랜덤 변동 (프론트 로컬) | 249 | 백엔드에 활성 세션 수 API 추가 후 SSE 또는 5분 폴링으로 조회 |
| 사용자 분포 파이차트 | 400, 300, 300 고정값 | 265-269 | `/admin/stats`에 구독 유형별 사용자 수 필드 추가 후 연동 |
| 화장실 트렌드 스파크라인 | `[12,15,18,22,25,28,32]` 고정 배열 | 276 | `/admin/stats`의 weeklyTrend에 일별 화장실 방문 수 추가 후 연동 |

## 2. SystemView (시스템 탭) — 전부 더미

> **참고**: 이 탭은 17개 지표가 전부 하드코딩이므로, 실제 연동 전까지 탭 자체를 "준비 중"으로 표시하는 것도 방법입니다.

| 항목 | 현재 상태 | 위치 (줄) | 개선 방향 |
|---|---|---|---|
| 시스템 성능 지표 6개 | UI 120/150, 성능 135/150 등 고정 | 2413-2420 | 백엔드 Actuator metrics 연동 또는 탭 비활성화 |
| 서버 상태 4개 | 가동시간 99.98%, 응답시간 14ms, 활성사용자 1,247, 분당요청 8.5K | 2422-2427 | 백엔드 Actuator health endpoint 연동 |
| 서비스 상태 4개 | DB, 레디스, API 게이트웨이, 파일 저장소 전부 "green" | 2550-2555 | 백엔드 Actuator health 컴포넌트별 상태 조회 |
| 하드웨어 지표 3개 | CPU 32.5%, 메모리 8.2GB/16GB, 디스크 245MB/s | 2577-2605 | 백엔드 Actuator system.cpu, jvm.memory 메트릭 연동 |

## 우선순위 제안

| 순위 | 작업 | 난이도 | 비고 |
|---|---|---|---|
| 1 | LogsView → 백엔드 `/admin/logs` API 연동 | 낮음 | API 이미 존재, fetch 추가만 필요 (백엔드 더미 수정 후) |
| 2 | 사용자 분포 파이차트 실제 데이터 연동 | 중간 | 백엔드에 구독 유형별 집계 API 추가 필요 |
| 3 | 현재 접속자 실시간 연동 | 중간 | 백엔드에 세션 추적 로직 구현 필요 |
| 4 | 화장실 트렌드 실제 데이터 연동 | 중간 | 백엔드에 일별 방문 집계 쿼리 추가 필요 |
| 5 | SystemView 서버 모니터링 연동 | 높음 | Actuator 설정 + 보안 + 프론트 연동 전체 작업 필요, 또는 탭 비활성화 |
| 6 | StoreView 테스트 버튼 운영 환경 숨김 | 낮음 | 환경 변수 분기 처리 |
