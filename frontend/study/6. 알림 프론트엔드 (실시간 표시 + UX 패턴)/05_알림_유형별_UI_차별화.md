# 섹션 5: 알림 유형별 UI 차별화 (디자인 시스템)

**[정합성 검증 완료 사항]**

> 사용자가 제시한 7종의 다채로운 도메인은 현재 코드베이스 상으론 간소화되어 있습니다. 실제 코드(`NotificationPanel.tsx` 및 `NotificationToast.tsx`)에서는 백엔드 Enum과 매핑하여 핵심 3~5가지 스타일로 렌더링하고 있습니다.

## 1. NotificationPanel 내부 아이콘 추출 로직

알림 패널 내 렌더링 시, 각 알림의 성질에 맞춰 시각적인 컬러 배색과 `lucide-react` 아이콘을 통일감 있게 매핑하는 구조입니다.

```tsx
import { Info, Trophy, MessageSquare } from 'lucide-react';

const getNotificationIcon = (type: string) => {
  switch (type) {
    case 'ACHIEVEMENT': // 칭호, 연속 기록 등
      return {
        Icon: Trophy,
        gradient: 'from-amber-500 to-orange-500',
        bg: 'bg-amber-50',
      };
    case 'SYSTEM': // 관리자 메시지, 문의 답변 등
    case 'EMERGENCY': // 긴급 공지 등
      return {
        Icon: MessageSquare,
        gradient: 'from-blue-500 to-cyan-500',
        bg: 'bg-blue-50',
      };
    case 'HEALTH': // 배변 패턴 리포트 알림
    case 'SOCIAL': // 친구 추가, 랭킹 변동 등
    default:
      return {
        Icon: Info,
        gradient: 'from-emerald-500 to-teal-500',
        bg: 'bg-emerald-50',
      };
  }
};

// ... JSX 렌더링 부 ...
const { Icon, gradient, bg } = getNotificationIcon(n.type);

<div className={`w-12 h-12 rounded-2xl flex items-center justify-center shrink-0 ${bg}`}>
  <div className={`bg-gradient-to-br ${gradient} bg-clip-text text-transparent`}>
    <Icon size={24} strokeWidth={2.5} />
  </div>
</div>;
```

## 2. NotificationToast 내부 UI 차별화 전략

`NotificationContext`를 통해 전역 프롬프트에서 떨어지는 `Toast` 객체는 `action` 상태(`error`, `star` 등)가 조금 더 다양하게 세팅되어 있습니다.

```tsx
import { X, Trophy, MessageSquare, Info, Star } from 'lucide-react';
export type ToastType = 'info' | 'achievement' | 'message' | 'star' | 'error';

// 아이콘 추출
const getIcon = (type: ToastType) => {
  switch (type) {
    case 'achievement':
      return <Trophy className="text-yellow-500" size={18} />;
    case 'message':
      return <MessageSquare className="text-blue-500" size={18} />;
    case 'star':
      return <Star className="text-purple-500 fill-purple-500" size={18} />;
    case 'error':
      return <X className="text-red-500" size={18} />;
    default:
      return <Info className="text-emerald-500" size={18} />;
  }
};

// 측면 띠(Accent Bar) 및 배경 호버(Hover) 컬러 추출
const getAccentColor = (type: ToastType) => {
  switch (type) {
    case 'achievement':
      return '#E8A838'; // Gold
    case 'message':
      return '#3B82F6'; // Blue
    case 'star':
      return '#A855F7'; // Purple
    case 'error':
      return '#EF4444'; // Red
    default:
      return '#1B4332'; // Dark Emerald (Brand Identity)
  }
};
```

---

> [!TIP] UX 설계 팁
> **형태(Icon)와 색상(Color)의 이중 인지 (Dual Coding Model)** <br>
> 텍스트를 읽지 않아도 사용자가 **초록색 띠 + 돋보기**를 보면 정보임을 알고, **빨간색 띠 + X 표**를 보면 에러임을 자동 반사적으로 깨닫게(Pre-attentive Processing) 만드는 것이 훌륭한 UI/UX입니다. 현재 코드의 `ToastType` 매핑은 TailwindCSS 기반으로 이러한 이중 인지 체계를 완벽하게 구축해 두었습니다.
