## 상세 내용

### #1. [Bug] 모바일 지도 드래그 불가

- **위치:** 지도 화면 (모바일 환경)
- **증상:** 모바일 브라우저에서 지도를 터치하여 드래그(패닝)할 수 없음
- **기대 동작:** 모바일에서도 터치 드래그로 지도 이동이 가능해야 함
- **비고:** 데스크톱 환경에서는 정상 동작하는지 확인 필요. 터치 이벤트 핸들링 또는 CSS `touch-action` 속성 확인 요망

---

### #2. [Bug] 방문 인증 시간 계산 로직 오류

- **위치:** 방문 인증 기능
- **현재 동작:** "방문 인증" 버튼을 누른 시점부터 체류 시간을 계산
- **기대 동작:** 해당 위치에 진입한 시점부터 체류 시간을 계산해야 함
- **수정 방향:**
  - 위치 진입 시점을 별도로 기록 (예: Geofencing 진입 이벤트 타임스탬프)
  - 방문 인증 요청 시, 버튼 클릭 시간이 아닌 진입 시점 기준으로 체류 시간을 산출하도록 로직 변경

---

### #3. [Bug] 방문 인증 종료 시 알림 연결 실패

- **위치:** 방문 인증 종료 처리
- **증상:** 방문 인증 완료(종료) 시 아래 에러 메시지가 노출됨
  > "알림 서비스에 연결할 수 없습니다. 나중에 다시 시도해주세요."
- **영향:** 사용자에게 인증 완료 알림이 전달되지 않음
- **확인 필요 사항:**
  - 알림 서비스(Notification Service)의 상태 및 연결 설정 확인
  - 방문 인증 종료 → 알림 발송 사이의 API 호출 흐름 점검
  - 이슈 #13(SSE 연결 실패)과의 연관성 확인

- **현재 동작:** 검색 결과가 지도 위 마커로만 표시됨
- **개선 요청:** 검색 결과를 모달(Modal) 또는 사이드바(Sidebar) 형태로 목록 제공

### #5. [Bug] 후기 무한 등록 가능

- **위치:** 후기 작성 기능
- **증상:** 동일 사용자가 동일 장소에 대해 후기를 제한 없이 반복 등록할 수 있음
- **영향:** 데이터 오염, 악용 가능성
- **수정 방향:**

### #11. [Bug] 로그인 후 "무료 리포트 시작하기" 클릭 시 재로그인 요구

- **위치:** 메인(Main) 화면 > "무료 리포트 시작하기" 버튼
- **증상:** 정상 로그인 완료 후 메인 화면에서 해당 버튼 클릭 시 로그인 화면으로 다시 이동됨
- **추측 원인:**
  - 해당 버튼의 링크가 인증이 필요한 경로를 가리키나, 토큰/세션이 전달되지 않는 경우
  - 로그인 상태 확인 로직의 오류 (토큰 갱신 타이밍 이슈)
- **확인 필요 사항:**
  - 버튼 클릭 시 호출되는 엔드포인트의 인증 헤더(Authorization) 포함 여부
  - 로그인 후 토큰 저장 및 전달 흐름 점검

### #12. [Bug] 아이템 구매 시 HTML 태그가 원시 텍스트로 노출

- **위치:** 마이페이지 > 컬렉션 > 아이템 상점 > 구매 액션
- **증상:** 아이템 구매 시 좌측 하단 알림(Toast)에 HTML 태그가 파싱되지 않은 채 그대로 노출됨
  ```
  <h3 class="text-sm font-black text-[#1A2B27] leading-tight mb-1">알림 연결 실패</h3>
  ```
- **문제점:**
  1. HTML이 이스케이프 없이 원시 문자열로 렌더링됨 (XSS 취약점 가능성)
  2. 알림 내용 자체가 "알림 연결 실패"로, 구매 성공/실패와 무관한 메시지임
- **수정 방향:**
  - Toast 컴포넌트에서 HTML 문자열이 아닌 순수 텍스트 또는 안전하게 렌더링되도록 수정
  - 구매 결과에 맞는 적절한 알림 메시지로 변경
  - 이슈 #13(SSE 연결 실패)과 동일 원인일 가능성 높음, 함께 확인 요망

### #13. [에러 리포트] SSE 연결 실패 (아이템 상점)

- **위치:** `mypage?tab=collection&sub=shop`
- **증상:**
  - `EventSource`가 `text/event-stream` 대신 `text/html` 응답을 수신
  - SSE 연결이 즉시 종료된 후 10회 재시도를 반복
- **에러 메시지:**
  ```
  EventSource's response has a MIME type ("text/html") 
  that is not "text/event-stream". Aborting the connection.
  ```

#### 추측 원인 (우선순위 순)

| 순위 | 원인 | 설명 |
|------|------|------|
| 1 | Nginx 프록시 버퍼링 | SSE 응답이 버퍼링되면서 HTML로 변환되어 전달 |
| 2 | Spring Security 미허용 | SSE 엔드포인트가 인증 필터에 막혀 302/401 응답 후 HTML 리다이렉트 |
| 3 | 컨트롤러 produces 누락 | `@GetMapping`에 `produces = TEXT_EVENT_STREAM_VALUE` 미설정 |
| 4 | 엔드포인트 404 | 경로 자체가 존재하지 않아 HTML 에러 페이지 반환 |

#### 확인 요청 사항

**브라우저 개발자 도구 > Network 탭에서 SSE 요청의 실제 Status Code를 확인해 주세요.**

| Status Code | 조치 방향 |
|-------------|----------|
| **404** | SSE 엔드포인트 URL 경로가 올바른지 확인 |
| **200** (MIME 오류) | Nginx 설정 또는 컨트롤러 `produces` 설정 확인 |
| **302 / 401** | Spring Security 필터에서 SSE 경로 허용 여부 확인 |

#### 참고: Nginx SSE 권장 설정

```nginx
location /api/sse {
    proxy_buffering off;
    proxy_cache off;
    proxy_set_header Connection '';
    proxy_http_version 1.1;
}
```
