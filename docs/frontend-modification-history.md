  - **`App.tsx` (로그인 라우트 수정):**
    - 백엔드에서 OAuth 인증 실패나 설정 누락 시 302 리다이렉트를 통해 프론트엔드의 `/login` 경로로 튕겨낼 때, 기존의 백지("로그인 페이지") 컴포넌트가 뜨지 않게 변경했습니다.
    - `/login` 접속 시 알림 처리 후 `<Navigate to="/main?login=open" />`를 통해 메인 맵 페이지에서 모달을 다시 열어주도록 수정했습니다. `useEffect`를 도입해 쿼리 파라미터를 감지하여 에러 여부를 안내합니다.
  - **`apiClient.ts` (JSON 파싱 에러 수정):**
    - `response.headers.get('content-type')`이 `application/json`임에도 백엔드(e.g., AuthController)가 순수 문자열만 내려주어 발생하는 `Unexpected token` 오류를 해결했습니다.
    - 파싱 전 문자열을 읽고, `JSON.parse()`가 실패할 경우 `try-catch`로 에러를 잡아 원본 문자열을 그대로 반환하도록 방어 로직을 추가했습니다.
- **결과/영향:** 이제 OAuth 카카오/구글 로그인이 실패(EC2 비밀키 누락 등)하더라도 백지 화면 대신 메인 화면에서 명확한 안내창을 받을 수 있습니다. 또한 이메일 중복 확인("사용 가능한 이메일입니다.") 등 텍스트 응답을 처리할 때 발생하던 파싱 오류가 사라져 정상적으로 기능이 작동합니다.

    - **토큰 리프레시 뮤텍스(Mutex):** 여러 요청이 동시에 401 에러를 발생시킬 때, 단 한 번만 토큰 갱신 요청을 보내고 나머지 요청은 그 결과를 기다려 재사용하도록 뮤텍스 패턴을 적용했습니다.
  - **SSE 연결 안정화 (`NotificationSubscriber.tsx`):** 알림 수신용 SSE 연결이 끊겼을 때, 최대 10회까지 지수 백오프 방식(최대 30초 지연)으로 재연결을 시도하여 실시간 알림의 연속성을 확보했습니다.
  - **환경 변수 정비 (`frontend/.env`):** 외부 API 연동을 위한 `VITE_API_URL` 환경 변수 설정을 추가 및 정비했습니다.
